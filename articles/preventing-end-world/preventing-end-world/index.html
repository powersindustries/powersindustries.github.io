Preventing the Collapse of Civilization - Jonathan Blow

May 17, 2021


In this famous lecture, Jonathan Blow gives a chilling warning to all software engineers. From a conference in Russia, Blow describes how past civilizations had technologies that were lost to time because the secrets of how to maintain these technologies were not passed down to the next generation of engineers. To solidify his point, Blow turns to the United States vs Soviet Union space race as a fitting example. Blow theorizes that the reason the world has yet to travel back to space is not because it is expensive, but because humanity probably does not remember how to.

After shocking the crowd, Blow turns to problems with modern software. According to Blow, the quality of software today has decreased compared to software of the past. The amount of bugs the average user puts up with is astonishing compared to older software.

Blow believes current software engineers heavily rely on the advancements in hardware to make today’s software work. In the past, software engineers were constantly battling the hardware limitations of the machines they were programming for. Today, engineers work with almost unlimited resources in both hardware and information. If a program runs out of memory, just dip into some virtual memory. If a programmer has any questions, a quick Stack Overflow search is the answer. Similar to veterans hardened by wartime conflicts, the strict hardware limitations of the past required the engineers working with them to come up with extremely efficient and creative solutions. This in-turn made these engineers better.

Towards the end of the lecture, Blow explains how modern software engineers can undo the damage that has been done. To reverse our impending doom, today’s software engineers need to go back and learn lower level languages and processes. After teaching themselves, they need to teach the next generations this information so that humanity does not forget. Many important systems like banks, payroll, weapon systems, etc all run on legacy languages and infrastructures that modern engineers no longer learn.

After completing the video for the first time, I sat for a few moments as the screen turned to black. Scrolling down to the comment section (never a good idea), many of the comments said things like, “Old man shakes fist at cloud,” along with other poorly written insults. Meanwhile, I sat in silence thinking about what I just heard. After letting Blow’s words soak in, I realized something.

Jonathan Blow is right.

Today, most software engineers learn specific languages with the hopes of getting a job. With the world becoming online, the current software engineering market is filled with high paying web developer positions. Most websites runs on high level languages like JavaScript, HTML/CSS, Python, and C#. Software engineers can leave school, accept a web developer position, and completely miss out on learning low level principles like memory management, cpu usage, and even project setup. All of these high level languages take care of these low level features out of sight from the developer. On top of this, many of these languages require package managers and external libraries to run at all. Instead of learning how to code, many modern engineers learn the quirks of their language environment instead of universal engineering principles.

For the future, we need to be better. If you are a modern engineer and Jonathan Blow has gotten to you, do not be disheartened. It is never too late to make a change.

For starters, modern engineers need to go back and learn basic engineering principles. The best way to learn is by doing. Modern software engineers need to learn at least one low to middle level programming language. I’m not saying to go nuclear and start learning assembly language or machine code (I currently am learning assembly language), but pick up a new language that requires the user to manage their own resources. Most new software engineers learned to code using scripting languages like JavaScript or Python. While they have their place, these scripting languages wont teach the user how to manage their own resources.

On top of this, modern engineers need to stop being lazy. In the video, Blow shows a small clip of Ken Thompson talking about how he created the Unix operating system. Users needed to be able to operate a computer easier, so Thompson spent three weeks of his own time and created Unix from scratch. Thompson did not use any off the shelf libraries or dependencies. He did not clone a Github repo or grab boiler plate code from a website. Instead, he created an editor, assembler, and an operating system each in one week successions.

I believe that engineers today should adopt this Ken Thompson mindset. We need to stop relying on other people’s code and do everything for ourselves. While it is convent to adopt someone else’s code for our own purposes, we miss out on the learning opportunity we could have had. At the cost of our own time, we lose out on gaining fundamental knowledge by accepting the Github handout. I believe the upfront cost of time is worth it in the end.

It is never too late to make a change. It is time for us to take control of our destiny and prevent the collapse of civilization.
