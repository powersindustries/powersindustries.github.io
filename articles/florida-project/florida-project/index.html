The Florida Project

Feb 1, 2023


A few months ago, I went on vacation to visit family in Florida. During the trip, I submitted the initial commit for a Git repo called the “Florida Project.” The goal of the project was to make a simple game using the MarkTwo engine. Eventually, the Florida Project turned into the game you see before you.

While working on a previous project, one of the biggest stressors for the team was that we were creating a game and its engine simultaneously. While this created a lot of uncertainty at the time, looking back, I realize that this created great art. While creating the game, we realized what features the engine needed. While making the engine, we came up with new gameplay features that could be added to the game. In the end, we needed both to happen simultaneously to make the other better. As they say, “Steel sharpens steel.”

In the same way, I realized that in order to make the MarkTwo engine better, it had to be stress tested against a game. Up until this point, the MarkTwo Engine was created in a vacuum. In order to make sure that the MarkTwo engine would work in a real-world environment, I needed to create a game to test MarkTwo with. Ultimately, the Florida Project made MarkTwo better and MarkTwo made the Florida Project better.

With the background out of the way, let’s talk about the game itself.

How to Play.

The Florida Project is a survival crafting game where the goal is to survive as many days as possible. The game operates on a day/night cycle. During the day, the player is free to move around the map safely. At night, enemies swarm and try to eat the player.

The player fights back using their trusty hammer and bow. The hammer is a close ranged attack that can hit multiple enemies. The bow shoots fast moving projectiles that kill enemies from a distance.

As enemies die, they periodically drop resources. The three main resources are scrap, wood, and water. These resources can be used to craft arrows and stamina potions. Arrows are the ammo use by the bow and stamina potions are used to remove the “Burn Out” effect players experience after depleting their stamina bar. Arrows and stamina potions can be used to get the player out of a bind when cornered by enemies.

Disclaimers.

From the beginning, I need to say that this game is not commercially viable in any way. The game assets look and sound terrible, the game loop is not fun, and the scope is limited. After all, I am an engineer.

That being said, the purpose of the Florida Project is not to be a good game, but to show that it is possible to create a game using the MarkTwo engine. Despite the game itself being bad, I believe the underlying systems are done properly. The Florida Project can be used as a reference when creating your own games with the MarkTwo engine.

The Florida Project includes examples of:

Core System Features

  • Input management system
  • Sound management system
  • Saved settings management system
  • UI system
  • Event system
  • Texture, font, and sound asset pipeline
  • System logging, hashing, and math functions
  • Centralized game loop
  • Map, enemy, and player data loading from xml data files

Gameplay Features

  • 2D melee and ranged combat
  • 2D pixel graphics
  • Day/Night cycle
  • Crafting system
  • Player movement and attack system
  • Enemy movement and attack system
  • UI screens for the Main menu, Pause menu, Crafting menu, HUD, and more
  • Poorly made 2D sprites and audio files
